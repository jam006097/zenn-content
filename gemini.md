# Zenn記事執筆規約

## 1. はじめに
このドキュメントは、Zennに記事を投稿する際のガイドラインを定めます。記事の品質を統一し、執筆作業を効率化することを目的としています。

## 2. ファイル管理規約
### 2.1 記事ファイル
- `articles` ディレクトリに配置してください。
- ファイル名は、記事のURLスラッグとなるように `[slug].md` の形式で記述してください。（例: `how-to-use-gemini-cli.md`）

### 2.2 画像ファイル
- `images` ディレクトリ内に、記事のファイル名（スラッグ）と同じ名前のサブディレクトリを作成し、その中に画像を配置してください。
    - 例: `images/how-to-use-gemini-cli/image1.png`
- 画像ファイルの命名規則は、半角英数字とハイフンのみを使用してください。

## 3. 記事のメタ情報 (Frontmatter)
各記事ファイルの先頭に、以下のFrontmatterを記述してください。

```yaml
---
title: "記事のタイトル" # 記事のタイトルを記述
emoji: "✨" # アイキャッチとなる絵文字を一つ記述
type: "tech" # "tech" (技術記事) または "idea" (アイデア記事) のいずれかを記述
topics: # 関連するタグを記述（推奨3〜5個、最大5個まで）
  - "gemini"
  - "cli"
  - "markdown"
published: false # 公開する場合は true, 下書きの場合は false
---
```

- `title`: 記事のタイトルを記述します。
- `emoji`: 記事一覧やSNSシェア時に表示されるアイキャッチとなる絵文字を一つ記述してください。
- `type`: 記事の種類を指定します。
    - `tech`: プログラミングや技術解説など、技術的な内容を扱います。
    - `idea`: 考え方、ノウハウ、学習方法など、アイデアや知見を共有する内容を扱います。
- `topics`: 記事の内容に関連するタグを記述してください。推奨は3〜5個で、最大5個までとします。
- `published`: 記事の公開状態を制御します。
    - `true`: 記事は公開されます。
    - `false`: 記事は下書き状態となり、公開されません。

## 4. 執筆スタイル・フォーマット
記事本文の品質を保つため、以下のルールを遵守してください。

### 4.1 文体
- 記事全体の文体は「ですます調」に統一してください。

### 4.2 表記
- 英数字は半角、カタカナは全角を使用してください。
- よく使われる技術用語や固有名詞（例: `Gemini`, `JavaScript`, `VS Code`, `GitHub`など）は、公式表記に従い、表記ゆれがないようにしてください。

### 4.3 Markdown記法
- 見出しは `#` (h1) を記事タイトルに使用し、本文中のセクションには `##` (h2) から使用してください。
- コードブロックを記述する際は、必ず言語指定を行ってください。
    - 例:
    ```typescript
    console.log("Hello, Zenn!");
    ```
- Zenn独自の記法（メッセージブロック、アコーディオンなど）の積極的な活用を推奨します。

## 5. 執筆・公開フロー
記事の作成から公開までの基本的な流れは以下の通りです。

1.  **記事ファイルの作成**:
    ```bash
    npx zenn new:article
    ```
    - このコマンドで自動生成されたファイルを元に執筆を開始します。

2.  **執筆**:
    - 上記の「ファイル管理規約」「記事のメタ情報」「執筆スタイル・フォーマット」に従って記事を執筆してください。

3.  **プレビュー**:
    - ローカルで記事の表示を確認してください。
    ```bash
    npx zenn preview
    ```
    - ブラウザで `http://localhost:8000` にアクセスして確認できます。

4.  **Gitへのコミット**:
    - 記事ファイルの追加・変更が完了したら、Gitにコミットしてください。
    - コミットメッセージは、以下の規約を参考にしてください。
        - 新規記事の追加: `feat(article): 新規記事「〇〇」を追加`
        - 既存記事の修正: `fix(article): 記事「〇〇」の誤字を修正`
        - 記事の更新: `docs(article): 記事「〇〇」の内容を更新`

5.  **公開**:
    - `main` ブランチにマージされると、Zennによって自動的に記事がデプロイ・公開されます。
    - もしくは、手動で `npx zenn publish` コマンドを使用することも可能です。
