# Zenn 記事執筆ガイドライン

## 【問題提起】なぜ、執筆ガイドラインが必要なのでしょうか？

記事を執筆する際、特にチームで作業する場合や、長期的にコンテンツを管理する場合において、いくつかの課題が生じることがあります。

- **一貫性の欠如**: 記事ごとにスタイルやフォーマットが異なると、読者が読みにくさを感じ、ブランド全体の統一感が損なわれる可能性があります。
- **品質のばらつき**: ルールが不明確な場合、記事の品質が執筆者個人のスキルや意識に依存してしまいます。
- **メンテナンス性の低下**: 将来的に記事を修正・更新する際、独自のルールで書かれていると、他の人が内容を理解しにくく、改修コストが高くなってしまいます。

これらの課題は、読者体験の低下や、将来の運用効率の悪化に繋がる可能性があります。

## 【結論】ガイドラインの遵守による、品質と効率の向上

そこで本ガイドラインは、一貫性のある高品質な記事を効率的に作成するための基準を定めます。

これらのルールに従うことで、読者にとって分かりやすく、かつ執筆者にとってメンテナンスしやすいコンテンツ資産を構築することを目指します。

## 【詳細】推奨される執筆スタイルとルール

### 1. 記事の基本情報を整える (CLI & フロントマター)

記事の第一印象となるメタデータは、読者が記事を読むかどうかの判断に大きく影響します。

- **記事の作成とURL (`slug`):**
  新しい記事を作成する際は、`slug` を指定することを推奨します。URLの一部となるため、記事の内容が推測できる名前にすると、読者にも分かりやすくなります。

  ```bash
  # slugを指定して記事を作成
  npx zenn new:article --slug csharp-testing-introduction
  ```

- **プレビューによる確認:**
  執筆中は `npx zenn preview` コマンドを使い、表示崩れや誤字がないかを確認しながら進めましょう。

- **フロントマターの適切な設定:**
  記事のメタ情報を定義するフロントマターは、非常に重要です。以下の項目は必ず設定してください。

  ```yaml
  ---
  title: "【C#】テストコード入門：未来の自分を助けるための第一歩"
  emoji: "🚀"
  type: "tech"
  topics: ["csharp", "testing", "beginner"]
  published: false # 公開準備が整うまで false に設定します
  ---
  ```
  `emoji` や `title` は読者の関心を引く要素であり、`topics` は記事を見つけてもらうために不可欠です。

### 2. 読者に配慮した本文の執筆

分かりやすく、読みやすい文章を心がけることが、読者の離脱を防ぎます。

- **文体:**
  「です・ます調」を基本とし、丁寧で分かりやすい言葉を選びましょう。必要に応じて絵文字を使い、親しみやすさを出すことも有効です。

- **コードブロックの書式:**
  コードを提示する際は、言語を指定してシンタックスハイライトを有効にしてください。これにより、コードの可読性が大幅に向上します。

  ````markdown
  ```csharp:BusinessDayCalculator.cs
  // このコードが何をしているのかをコメントで補足すると、より親切です
  public class BusinessDayCalculator
  {
      // ...
  }
  ```
  ````

- **Zenn独自記法の活用:**
  `:::message` などのカスタムブロックを活用すると、補足情報や注意点を効果的に伝えられます。

### 3. 画像の適切な管理

記事の信頼性を損なわないよう、画像は適切に管理します。

- **配置場所:**
  画像ファイルは `/images` ディレクトリ以下に、記事のテーマに合わせたサブディレクトリを作成して管理すると、後から見返しやすくなります。
- **パスの指定:**
  画像を指定する際は、プロジェクトルートからの絶対パス (`/images/...`) を使用してください。これにより、ファイルの移動などによるリンク切れを防ぐことができます。

  ```markdown
  ![](/images/csharp/csharp-test-calcday-code.png)
  ```

## 【まとめ】品質への投資が、未来の資産となる

本ガイドラインに記載されたルールは、一見すると少し手間に感じられるかもしれません。
しかし、これらは記事の品質と一貫性を保ち、将来にわたってコンテンツを円滑に運用していくための重要な投資です。

今日行う少しの工夫が、明日のメンテナンスコストを削減し、読者の満足度を高めることに繋がります。
